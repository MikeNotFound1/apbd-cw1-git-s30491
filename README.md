\# Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?

Fast forward wykona się gdy branch docelowy nie ma żadnych commitów do momentu stworzenia drugiego brancha. Merge commit powstaje gdy obie gałęzie mają oddzielne własne commity i git musi połączyć dwie linie historii w jedną.

\#Czym w praktyce różni się merge od rebase?

Merge łączy dwie historie w jedną, zachowując przy tym wszystkie informacje. Rebase łączy dwie historie w jedną, bez zachowania informacji o drugim branchu co powoduje, że historia wygląda na liniową.
